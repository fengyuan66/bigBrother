An agent has access to a to-do-list, memory, and status file

To-do-list allows agents to basically programmatically set an alarm (a stimuli or call) for themselves (with notes attached) using simple syntax

Memory is a timestamped list of user movements and other notes. The Agent doesn't have to reference it, but it can use RAG to reference it at its choosing

A status file is more recommended to frequently access. This contains the current state of things and status of the user and their activity, including those derived from what the user said in response to (for ex, ) questions. The Agent can change this in the loop, as indicated in the diagram
